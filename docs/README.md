## Decision Records

ADRs under [`docs/adr/`](adr/) record durable architectural decisions. Accepted ADRs are authoritative; proposed decisions must not be described as accepted architecture until their status changes.

The OpenAPI document generated by the running backend at `/docs` is the authoritative endpoint schema. Narrative documentation explains workflows and boundaries without replacing that generated contract.
